# Render 클라우드 배포 가이드

이 가이드대로 따라하면 **PC가 꺼져 있어도 모바일에서 24시간 접속 가능**한 클라우드 서버에 본 앱이 배포됩니다.

## 0. 사전 준비

- GitHub 계정 (https://github.com)
- Render 계정 (https://render.com — 무료, GitHub 계정으로 가입 가능)

## 1. GitHub에 코드 업로드

### 1-1. 새 저장소(repository) 만들기
1. https://github.com/new 접속
2. Repository name: `thermal-report` (원하는 이름)
3. Private 선택 (회사 내부 코드라서)
4. **Create repository** 클릭

### 1-2. 폴더 통째로 업로드
1. 새로 만든 저장소 페이지에서 **uploading an existing file** 클릭
2. **web_app 폴더 안의 모든 파일과 폴더**를 드래그앤드롭
   - templates/, static/, templates_pdf/, app.py, report_engine.py, flir_decode.py, requirements.txt, Procfile, render.yaml 등 전부
   - `reports.db`, `__pycache__`, `uploads/`, `reports/` 안의 임시 파일은 제외
3. **Commit changes** 클릭

## 2. Render에서 Web Service 생성

### 2-1. Render 가입/로그인
1. https://render.com 접속 → **Get Started for Free** → GitHub 계정으로 가입
2. 좌측 메뉴 **New +** → **Web Service**

### 2-2. GitHub 저장소 연결
1. **Connect a repository** → 방금 만든 `thermal-report` 선택
2. 처음이면 GitHub 권한 승인 필요

### 2-3. 서비스 설정
| 항목 | 값 |
|---|---|
| Name | `thermal-report` (URL이 됨: thermal-report.onrender.com) |
| Region | Singapore (한국에서 가장 빠름) |
| Branch | main |
| Runtime | Python 3 (자동 감지) |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --threads 4 --timeout 120` |
| Instance Type | **Free** |

### 2-4. 환경 변수(Environment Variables) 추가
**Advanced** → **Add Environment Variable** 클릭하여 다음 3개 추가:

| Key | Value | 비고 |
|---|---|---|
| `APP_PASSWORD` | (예: `kcc2026!`) | 로그인 시 입력할 비밀번호 — 회사 내부 공유용 |
| `SECRET_KEY` | (랜덤 문자열, 32자 이상) | 세션 암호화 키. 아무거나 길게 입력. 예: `aB3xK9pQ7mN2vR5sL8wT4yU1jH6gF0dE` |
| `DATA_DIR` | `/var/data` | 영구 저장 디스크 마운트 위치 |

### 2-5. Persistent Disk 추가 (보고서 DB + 업로드 사진 저장용)
**Advanced** → **Add Disk** 클릭:

| 항목 | 값 |
|---|---|
| Name | `thermal-data` |
| Mount Path | `/var/data` |
| Size | `1 GB` (무료) |

### 2-6. 배포 시작
**Create Web Service** 클릭 → 자동으로 빌드/배포 시작 (5~10분 소요)

## 3. 접속 확인

배포 완료 후 화면 상단의 URL (예: `https://thermal-report.onrender.com`) 클릭:
1. 로그인 화면이 뜨면 정상
2. APP_PASSWORD 입력 → 메인 화면 진입
3. 현장 선택 → 측정값 입력 → 저장 / PDF 생성 모두 작동

## 4. 모바일에서 접속

폰 브라우저(Chrome / Safari)에서 위 URL 접속:
1. 로그인 화면 → APP_PASSWORD 입력
2. 한 번 로그인하면 30일간 자동 로그인 유지 (`PERMANENT_SESSION_LIFETIME=30일`)
3. PC가 꺼져 있어도 24시간 접속 가능

## 5. 주의사항 (무료 플랜)

- **15분 비활성 시 sleep**: 아무도 접속 안 하면 서버가 잠들고, 첫 접속 시 30초~1분 대기 (이후 빠름)
- **750시간/월**: 한 달 24시간 항상 켜둬도 무료 한도 안쪽
- **Disk 1GB**: 보고서 1000건 정도까지는 충분. 사진은 자동 리사이즈되어 저장됨

## 6. 코드 업데이트하는 방법

GitHub 저장소에 새 파일 업로드(또는 git push) → Render가 **자동 감지하고 재배포**.
- 약 3~5분 후 새 코드가 반영됨
- 환경변수는 그대로 유지됨

## 7. 비밀번호 변경하기

Render 대시보드 → 서비스 선택 → **Environment** → `APP_PASSWORD` 수정 → **Save Changes**.
저장하면 자동으로 재배포되어 새 비밀번호 적용됨.

## 8. 자주 발생하는 문제

### "Bad Gateway" 또는 빌드 실패
- Render Logs 확인 (대시보드 → Logs 탭)
- requirements.txt에 의존성 누락 확인
- Python 3.11 사용 (PYTHON_VERSION=3.11.7 환경변수 추가 가능)

### 로그인 화면이 안 뜸
- APP_PASSWORD 환경변수 누락 → 인증 우회 모드로 작동 (모두 접속 가능)
- 환경변수 추가 후 자동 재배포 기다리기

### DB 데이터가 없어짐
- Persistent Disk가 마운트 안 됐을 가능성 → Mount Path 확인 (반드시 `/var/data`)
- `DATA_DIR=/var/data` 환경변수 확인

### 사진이 큰 경우 업로드 실패
- 기본 100MB 제한. 더 큰 사진이면 모바일에서 자동 리사이즈 (MAX_SIDE=1280px) 후 업로드
